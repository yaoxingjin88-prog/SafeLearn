# 储能电站安全培训与事故推演系统 - 开发文档

## 1. 项目概述

### 1.1 项目名称
- **正式名称：** 储能电站安全培训与事故推演系统
- **备选品牌名：** 智储安 / EHS-Storage / 储安云 / ESS Safe

### 1.2 项目简介
面向储能电站场景的智能化安全培训与事故推演平台，通过事故动态推演、热失控模拟、AI安全问答、应急决策训练、可视化交互等功能，提升储能行业从业人员的风险辨识能力和应急处置能力。

### 1.3 项目目标
- 构建数字化、智能化、沉浸式的安全培训平台
- 实现储能热失控事故的动态推演与模拟
- 提供AI辅助的安全知识问答服务
- 建立应急决策交互训练体系
- 推动储能安全培训向数字化转型

### 4.3 应急决策训练模块

#### 功能描述
系统模拟事故场景，用户选择处置操作，系统实时评分并生成训练报告。

#### 决策树设计
```
场景：某储能舱温度达到85℃
│
├── 选择1：立即断电
│   ├── 是 → 降低热扩散风险 (+20分)
│   └── 否 → 继续升温，风险增加
│
├── 选择2：启动消防系统
│   ├── 是 → 控制火势 (+25分)
│   └── 否 → 火势蔓延
│
├── 选择3：疏散人员
│   ├── 是 → 保障人员安全 (+30分)
│   └── 否 → 人员伤亡风险
│
└── 选择4：通知消防部门
    ├── 是 → 获得外部支援 (+25分)
    └── 否 → 缺乏外部支援
```

#### 数据结构
```typescript
interface TrainingScenario {
  id: string;
  name: string;
  description: string;
  difficulty: 'easy' | 'medium' | 'hard';
  timeLimit: number; // 秒
  decisionPoints: DecisionPoint[];
}

interface DecisionPoint {
  id: string;
  scenarioId: string;
  triggerCondition: string;
  question: string;
  options: DecisionOption[];
  timePressure: number; // 倒计时秒数
}

interface DecisionOption {
  id: string;
  text: string;
  score: number;
  consequence: string;
  nextDecisionPointId?: string;
  isCorrect: boolean;
}

interface TrainingRecord {
  id: string;
  userId: string;
  scenarioId: string;
  startTime: Date;
  endTime: Date;
  decisions: {
    decisionPointId: string;
    selectedOptionId: string;
    responseTime: number; // 毫秒
    score: number;
  }[];
  totalScore: number;
  rating: 'excellent' | 'good' | 'average' | 'poor';
  feedback: string;
}
```

#### API接口
```typescript
// 获取训练场景列表
GET /api/training/scenarios
QueryParams: { difficulty?, page?, pageSize? }
Response: { scenarios: TrainingScenario[], total: number }

// 开始训练
POST /api/training/start
Body: { scenarioId }
Response: { recordId: string, firstDecisionPoint: DecisionPoint }

// 提交决策
POST /api/training/decision
Body: { recordId, decisionPointId, optionId, responseTime }
Response: { score, consequence, nextDecisionPoint? }

// 获取训练报告
GET /api/training/records/:id/report
```

## 5. 数据库设计

### 5.1 核心表结构

```sql
-- 用户表
CREATE TABLE users (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    username VARCHAR(50) UNIQUE NOT NULL,
    email VARCHAR(100) UNIQUE NOT NULL,
    password_hash VARCHAR(255) NOT NULL,
    role VARCHAR(20) NOT NULL DEFAULT 'trainee', -- admin, manager, trainee
    company VARCHAR(100),
    department VARCHAR(100),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 课程表
CREATE TABLE courses (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    title VARCHAR(200) NOT NULL,
    description TEXT,
    cover_image VARCHAR(500),
    category VARCHAR(50) NOT NULL,
    total_duration INTEGER, -- 分钟
    status VARCHAR(20) DEFAULT 'draft', -- draft, published, archived
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 章节表
CREATE TABLE chapters (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    course_id UUID REFERENCES courses(id),
    title VARCHAR(200) NOT NULL,
    content TEXT,
    video_url VARCHAR(500),
    duration INTEGER, -- 分钟
    order_num INTEGER NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 学习进度表
CREATE TABLE user_progress (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID REFERENCES users(id),
    course_id UUID REFERENCES courses(id),
    chapter_id UUID REFERENCES chapters(id),
    progress INTEGER DEFAULT 0, -- 0-100
    completed_at TIMESTAMP,
    last_access_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    UNIQUE(user_id, chapter_id)
);

-- 推演场景表
CREATE TABLE scenarios (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    name VARCHAR(200) NOT NULL,
    description TEXT,
    initial_conditions JSONB NOT NULL,
    events JSONB NOT NULL,
    duration INTEGER NOT NULL, -- 秒
    difficulty VARCHAR(20) DEFAULT 'medium',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 训练记录表
CREATE TABLE training_records (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID REFERENCES users(id),
    scenario_id UUID REFERENCES scenarios(id),
    start_time TIMESTAMP NOT NULL,
    end_time TIMESTAMP,
    decisions JSONB,
    total_score INTEGER,
    rating VARCHAR(20),
    feedback TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 事故案例表
CREATE TABLE accident_cases (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
```
